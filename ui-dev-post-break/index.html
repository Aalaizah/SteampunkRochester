UI Dev, post Break

05 April 2015

Programming team here is coming in strong after this break with nearly all of the core architecture taken care of. I was supposed to get a lot more work done this break, however I experienced an unexpected reaction to a shot that I had. It was literally the hit by the bus scenario. I was originally tasked with getting the choices to show up on the screen. I wasn’t able to do that, only being able to set up a majority of the logic necessary for said task to be completed. Thanks to Tyler’s debugging skills and knowledge of the parser, we were able to fix the issue.

From this point on now, I am learning about the UI system that was implemented in Unity 4.6.1. This will be the version of Unity that we will use, and the new UI system was a major overhaul of the previous version. On top of that, I am also working on wireframe sketches for the game’s UI, including menu, game, and scene views.

At this point I am looking forward to start seeing art assets populating our game, and to start scripting out the story.

Joe Coppola
